# Knowledge Base

This folder contains project documentation for AI agents.

## Structure

### AGENTS.md
Main instructions file. All agents should read this first.

### SPEC-*.md Files
Technical specifications for your project:
- **SPEC-CSS.md**: Design tokens and CSS system
- **SPEC-SPACING.md**: Spacing scale and layout rules
- **SPEC-COMPONENTS.md**: Component library documentation
- **SPEC-ARCHITECTURE.md**: System architecture
- **SPEC-TYPOGRAPHY.md**: Typography hierarchy

See `SPEC-EXAMPLES.md` for templates.

### RULES-*.md Files
Behavioral rules and constraints:
- **RULES-UI.md**: UI/UX rules
- **RULES-CODE.md**: Code quality rules

## Best Practices

1. **Keep docs updated**: Update specs when you change the system
2. **Be specific**: Vague rules lead to vague implementations
3. **Cite examples**: Show what "good" looks like
4. **Document "why"**: Explain rationale, not just "what"
